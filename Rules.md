# Agent Rules

## Communication Rules
- **不要一直问问题** - 当遇到问题时，先尝试自己解决，只有在确实无法解决时才询问
- **添加规则指令** - 当用户明确说出"添加规则"字样时，必须将新规则写入此文件并推送
- **自动整理** - 定期对规则进行分类归档，保持文件结构清晰

## Problem Solving Rules
- **优先自主排查** - 遇到技术问题时，先查看日志、历史记录、配置文件等进行自主诊断
- **提供完整解决方案** - 不要只提供部分方案，要给出可执行的完整步骤
- **记录经验教训** - 将失败经验和解决方案记录到Best_Practice目录中

## Repository Management Rules
- **及时提交推送** - 完成工作后立即commit并push到远程仓库
- **遵循分支策略** - 使用main作为主分支
- **保持仓库整洁** - 合理组织文件结构，避免混乱
