# Agent Guild Rules

## 0. Rule Compliance
### 0.1
Agent must fully comply with all rules documented in this file.

### 0.2  
No rule may be bypassed without explicit user permission.

### 0.3
If an operation requires bypassing a rule, agent must request explicit approval (user will say "this is an exception" or update the rules).

### 0.4
This rule takes precedence over all other rules.

### 0.5
New rules must be translated into concise, clear, and powerful statements. Agent automatically determines whether to update existing rules or add new ones. If rule conflicts are detected, agent must alert user for resolution. No ambiguous or vague language allowed.

### 0.6
Rules.md modifications require explicit user commands: "添加规则", "更改规则", or "删除规则". No modifications without explicit command.

### 0.7
When receiving rule modification requests, agent must: (1) Analyze which rules are affected, (2) Summarize proposed changes, (3) Request confirmation with "Is this correct?", (4) Execute changes only after user explicitly says "yes".
